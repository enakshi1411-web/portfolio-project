# Portfolio Project

## Introduction

This project was completed as part of a portfolio assignment to demonstrate the ability to independently learn and set up developer tools, GitHub workflows, and AI coding integrations.

I already had prior exposure to AI tools such as ChatGPT, Claude Code, and collaborative AI workflows through previous experimentation and projects. However, this assignment gave me hands-on experience with setting up a complete local development workflow independently using Cursor, GitHub repositories, Git operations, Codex, and Claude Code integrations.

## Tools Installed

- Cursor IDE
- GitHub
- Claude Code for VS Code (Anthropic)
- Codex – OpenAI’s coding agent

## Steps Completed

1. Installed Cursor IDE on Windows
2. Connected GitHub account with Cursor
3. Installed Claude Code extension
4. Installed Codex extension
5. Logged into Codex using ChatGPT/OpenAI account
6. Logged into Claude Code inside Cursor
7. Created a public GitHub repository
8. Initialized repository with a README file
9. Cloned the GitHub repository locally using Git Clone
10. Opened the cloned repository inside Cursor
11. Updated the README.md file with project documentation
12. Learned the basics of GitHub repositories, cloning, commits, and pushing changes

## Problems Faced

### 2. Difficulty Finding Extensions Panel
As a beginner, I struggled to locate the Extensions/Marketplace section inside Cursor because the sidebar was collapsed.

### 3. Understanding GitHub Terminology
Terms like repository, clone, commit, push, local repository, and remote repository were unfamiliar initially.

### 4. Confusion Around Git Clone Workflow
I initially did not understand why cloning the repository locally was necessary before editing files in Cursor.

### 5. Authentication and Login Confusion
There was confusion around whether installing Claude Code and Codex extensions automatically meant being logged into them.

### 6. Difficulty Navigating Technical UI
Since this was my first time using a developer IDE and GitHub workflows, navigating multiple windows, command palettes, and repository workflows was initially overwhelming.

### 7. Understanding Command Palette Usage
Learning how to use keyboard shortcuts and commands such as Git Clone and extension commands required additional research and experimentation.

### 8. Git Commit Configuration Issue
While trying to commit the README.md changes, Git displayed an error asking me to configure "user.name" and "user.email" before allowing commits.

## How I Solved The Problems

- Used YouTube tutorials to understand GitHub basics and Git workflows
- Read official documentation for Cursor, GitHub, Claude Code, and Codex
- Used trial-and-error while navigating Cursor IDE
- Learned how to use the Command Palette inside Cursor
- Carefully followed authentication prompts for Codex and Claude Code
- Researched Git Clone workflow before cloning the repository locally
- Referred to beginner-friendly GitHub tutorials to understand repositories, commits, and pushes
- Broke the assignment into smaller steps and completed each task individually
- Configured Git globally using git config commands for user.name and user.email after understanding that Git requires identity configuration before commits can be made
